## Core Concepts

### Basic Structure of Workflows

Understanding the basic structure of workflows is the foundation for creating any workflow. A complete workflow typically consists of three main parts:

#### 1. Trigger

A trigger is the starting point of a workflow, determining when the workflow starts. Trigger nodes are usually located on the leftmost side of the workflow.

**Common Trigger Types**:
- **Schedule Trigger**: Scheduled trigger that executes on a time schedule (e.g., every day at 9 AM, every Monday)
- **Webhook Trigger**: Webhook trigger that receives external HTTP requests
- **Email Trigger**: Email trigger that triggers when a new email is received
- **Manual Trigger**: Manual trigger that executes when manually clicked

**Functions of Triggers**:
- Start the workflow
- Provide initial data
- Determine when the workflow executes

#### 2. Processing Nodes

Processing nodes are the core of workflows, performing actual work: fetching data, transforming data, executing operations, etc.

**Common Processing Node Types**:
- **HTTP Request**: Send HTTP requests, call APIs
- **Set**: Set or modify data fields
- **Code**: Execute JavaScript code
- **IF**: Conditional judgment, branch based on conditions
- **Function**: Execute custom functions

**Characteristics of Processing Nodes**:
- Receive data from upstream nodes
- Process the data
- Pass processed data to downstream nodes

#### 3. Output Nodes

Output nodes are the endpoints of workflows, outputting results to external systems or saving data.

**Common Output Node Types**:
- **Email**: Send emails
- **Slack**: Send Slack messages
- **Google Sheets**: Write to Google Sheets
- **Webhook Response**: Return HTTP response
- **Save to File**: Save to file

**Workflow Structure Diagram**:
```
[Trigger] â†’ [Processing Node 1] â†’ [Processing Node 2] â†’ ... â†’ [Output Node]
```

### Adding and Connecting Nodes

#### Adding Nodes

In N8N, adding nodes is very simple:

1. **Click the "+" button on the canvas** or **double-click blank area on canvas**
2. **Search for node**: Enter node name in search box (e.g., "Schedule", "HTTP Request")
3. **Select node**: Choose the needed node from search results
4. **Node appears on canvas**: Node is automatically added to canvas

[Screenshot location: Interface for adding nodes]

#### Connecting Nodes

Connections between nodes define data flow:

1. **Drag from source node**: Place mouse on right connection point of source node
2. **Drag to target node**: Drag connection line to left connection point of target node
3. **Connection complete**: Connection line appears between two nodes

**Connection Rules**:
- Data can only flow from upstream nodes to downstream nodes
- A node can have multiple output connections (data branching)
- A node can have multiple input connections (data merging)

[Screenshot location: Diagram of node connections]

#### Configuring Nodes

Each node needs configuration to work properly:

1. **Click node**: Select node to configure
2. **Open configuration panel**: Right panel shows node configuration panel
3. **Fill parameters**: Fill corresponding parameters based on node type
4. **Save configuration**: Configuration is automatically saved

**Configuration Examples**:
- **Schedule Trigger**: Set execution time (e.g., "Every day at 9 AM")
- **HTTP Request**: Set URL, method, headers, etc.
- **Set**: Set fields to add or modify

### Workflow Execution and Testing

#### Testing Workflows

Before formally executing a workflow, it's recommended to test first:

1. **Click "Test workflow" button**: In workflow toolbar
2. **Select test node**: Can choose to test entire workflow or single node
3. **View test results**: Test results displayed below node
4. **Check data**: Click node to view input and output data

**Benefits of Testing**:
- Quickly verify if node configuration is correct
- View input and output data of each node
- Discover configuration errors, avoid execution failures

#### Executing Workflows

After testing passes, can formally execute workflow:

1. **Activate workflow**: Click switch in top right corner of workflow to activate
2. **Manual execution**: Click "Execute Workflow" button to manually execute
3. **Automatic execution**: If using trigger (e.g., Schedule), workflow will automatically execute
4. **View execution results**: View execution history in "Executions"

#### Execution Logs

Execution logs record the execution process of workflows:

- **Execution status**: Success (green), failure (red), running (yellow)
- **Execution time**: Start time, end time, duration
- **Node status**: Execution status and data of each node
- **Error information**: If failed, shows detailed error information

## Detailed Tutorial

### Step 1: Create Your First Workflow

Let's start by creating a simple workflow.

#### 1.1 Open N8N and Create New Workflow

1. **Login to N8N**: Visit http://localhost:5678 and login
2. **Create new workflow**:
   - Click "Workflows" in left sidebar
   - Click "Add workflow" button in top right
   - Or click "New workflow" button

[Screenshot location: Interface for creating new workflow]

#### 1.2 Name Workflow

1. **Click workflow name**: Default name is "Untitled Workflow"
2. **Enter new name**: e.g., "My First Workflow"
3. **Save**: Name is automatically saved

#### 1.3 Add First Node - Schedule Trigger

1. **Double-click blank area on canvas** or **click "+" button**
2. **Search "Schedule"**: Enter "Schedule" in search box
3. **Select "Schedule Trigger" node**
4. **Node appears on canvas**

[Screenshot location: Adding Schedule Trigger node]

#### 1.4 Configure Schedule Trigger

1. **Click Schedule Trigger node**
2. **Set in right configuration panel**:
   - **Trigger Times**: Select "Every minute" (execute every minute, for testing)
   - Or select "Every day at 9:00 AM" (execute daily at 9 AM)
3. **Configuration complete**: Configuration is automatically saved

**Configuration Notes**:
- **Every minute**: Execute every minute (suitable for testing)
- **Every hour**: Execute every hour
- **Every day at [time]**: Execute daily at specified time
- **Cron expression**: Use Cron expression to customize time

[Screenshot location: Schedule Trigger configuration panel]

### Step 2: Add HTTP Request Node to Fetch Data

Now let's add a node to fetch data.

#### 2.1 Add HTTP Request Node

1. **Click connection point on right side of Schedule Trigger node**
2. **Drag connection line**: Node selection menu appears
3. **Search "HTTP Request"**: Enter "HTTP Request"
4. **Select "HTTP Request" node**
5. **Node automatically connected**: HTTP Request node automatically connects to Schedule Trigger

[Screenshot location: Adding HTTP Request node]

#### 2.2 Configure HTTP Request Node

1. **Click HTTP Request node**
2. **Set in configuration panel**:
   - **Method**: Select "GET"
   - **URL**: Enter URL of a public API, for example:
     - Weather API: `https://api.openweathermap.org/data/2.5/weather?q=Beijing&appid=YOUR_API_KEY`
     - Or use free test API: `https://jsonplaceholder.typicode.com/posts/1`
     - Or use simple API: `https://api.github.com/zen` (returns random quote)
3. **Save configuration**

**Recommended Test APIs** (no API key needed):
- `https://jsonplaceholder.typicode.com/posts/1`: Returns sample JSON data
- `https://api.github.com/zen`: Returns random quote
- `https://httpbin.org/get`: Returns request information

[Screenshot location: HTTP Request configuration panel]

#### 2.3 Test HTTP Request Node

1. **Click "Test workflow" button** (in workflow toolbar)
2. **Select HTTP Request node**: Click "Test step" below node
3. **View results**: API returned data displayed below node
4. **Check data format**: Confirm returned data is JSON format

[Screenshot location: Test results]

### Step 3: Add Set Node to Process Data

Now let's add a node to process and format data.

#### 3.1 Add Set Node

1. **Click connection point on right side of HTTP Request node**
2. **Drag and search "Set"**
3. **Select "Set" node**
4. **Node automatically connected**

[Screenshot location: Adding Set node]

#### 3.2 Configure Set Node

Set node is used to add, modify, or delete data fields.

1. **Click Set node**
2. **Set in configuration panel**:
   - **Keep Only Set Fields**: Uncheck (keep original fields)
   - **Values to Set**: Click "Add Value"
     - **Name**: Enter field name, e.g., `message`
     - **Value**: Enter value or use expression
       - If using GitHub API, can enter: `{{ $json }}`
       - Or enter fixed value: `"Hello from N8N!"`
     - Add another field:
       - **Name**: `timestamp`
       - **Value**: `{{ $now }}` (current time)

3. **Save configuration**

**Expression Notes**:
- `{{ $json }}`: Get JSON data from upstream node
- `{{ $now }}`: Get current time
- `{{ $json.fieldName }}`: Get specific field from JSON

[Screenshot location: Set node configuration panel]

#### 3.3 Test Set Node

1. **Click "Test workflow" button**
2. **View Set node output**: Confirm data is correctly set

### Step 4: Add Output Node

Now let's add an output node to display results.

#### 4.1 Choose Output Method

Based on your needs, can choose different output methods:

**Method A: Use Webhook Response (Recommended for testing)**
- Simple, no additional configuration needed
- Suitable for testing and demonstration

**Method B: Use Email Node**
- Requires email service configuration
- Suitable for actual applications

**Method C: Use Slack Node**
- Requires Slack configuration
- Suitable for team notifications

#### 4.2 Add Webhook Response Node (Method A)

1. **Click connection point on right side of Set node**
2. **Search "Webhook Response"**
3. **Select "Webhook Response" node**
4. **Configure node**:
   - **Response Code**: 200
   - **Response Body**: `{{ $json }}` (return processed data)

[Screenshot location: Webhook Response node]

**Note**: Webhook Response needs to be used with Webhook Trigger. For Schedule Trigger, we can use other output methods.

#### 4.3 Add Code Node as Output (Alternative)

Since Schedule Trigger cannot directly use Webhook Response, we can use Code node to output results:

1. **Click connection point on right side of Set node**
2. **Search "Code"**
3. **Select "Code" node**
4. **Configure node**:
   - **Mode**: Run Once for All Items
   - **JavaScript Code**:
   ```javascript
   // Get all data
   const items = $input.all();
   
   // Output to console (view in N8N execution logs)
   console.log('Workflow executed successfully!');
   console.log('Data:', JSON.stringify(items, null, 2));
   
   // Return data
   return items;
   ```

5. **Save configuration**

[Screenshot location: Code node configuration]

### Step 5: Save and Execute Workflow

#### 5.1 Save Workflow

1. **Click "Save" button in toolbar** (or press Ctrl+S / Cmd+S)
2. **Workflow saved**: "Saved" prompt appears in top left

#### 5.2 Activate Workflow

1. **Click switch in top right corner of workflow**: Set workflow to "Active"
2. **Workflow activated**: Switch turns green

**Note**: Only activated workflows will automatically execute (if using Schedule Trigger).

#### 5.3 Manually Execute Workflow

1. **Click "Execute Workflow" button in toolbar**
2. **Wait for execution to complete**: Workflow will start executing
3. **View execution results**:
   - Each node shows execution status (green means success)
   - Click node to view input and output data

[Screenshot location: Execution results]

#### 5.4 View Execution History

1. **Click "Executions" in left sidebar**
2. **View execution list**: Shows all workflow execution records
3. **Click execution record**: View detailed execution logs
4. **Analyze execution process**:
   - View execution time of each node
   - View data changes at each node
   - If failed, view error information

[Screenshot location: Execution history interface]

### Step 6: Understand Data Flow

Understanding how data flows between nodes is very important.

#### 6.1 Data Format

N8N uses JSON format to pass data:

```json
{
  "json": {
    "field1": "value1",
    "field2": "value2"
  },
  "binary": {},
  "error": null
}
```

#### 6.2 Access Data

In nodes, can use expressions to access data:

- `{{ $json }}`: Get entire JSON object
- `{{ $json.fieldName }}`: Get specific field
- `{{ $json.array[0] }}`: Get array element
- `{{ $now }}`: Get current time
- `{{ $workflow.name }}`: Get workflow name

#### 6.3 Data Flow Tracking

1. **Execute workflow**
2. **Click each node**: View input and output data
3. **Compare data changes**: Observe data changes at each node
4. **Draw data flow diagram**: Record data state as it flows through each node

## Common Questions

**Q1: How to add nodes?**

A: Several ways:
1. Double-click blank area on canvas, search node name
2. Click "+" button on canvas
3. Drag connection line from source node, select target node

**Q2: What are the rules for connections between nodes?**

A:
- Data can only flow from upstream nodes to downstream nodes
- A node can have multiple output connections (data branching)
- A node can have multiple input connections (data merging)
- Cannot form circular connections (Node A â†’ Node B â†’ Node A)

**Q3: How to test a single node?**

A:
1. Click "Test workflow" button
2. Select node to test
3. Click "Test step" below node
4. View test results

**Q4: What to do if workflow execution fails?**

A:
1. View execution history, find failed execution record
2. Click failed node, view error information
3. Fix problem based on error information
4. Re-execute workflow

**Q5: How to view input and output data of nodes?**

A:
1. After executing workflow
2. Click node
3. View data below node or in right panel
4. Can expand JSON to view detailed data

**Q6: What does expression `{{ $json }}` mean?**

A: This is N8N's expression syntax:
- `{{ }}`: Expression wrapper
- `$json`: JSON data of current node
- `$json.fieldName`: Get specific field
- `$now`: Current time

**Q7: How to make workflow execute automatically?**

A:
1. Use trigger node (e.g., Schedule Trigger)
2. Activate workflow (click switch in top right)
3. Workflow will automatically execute when trigger condition is met

**Q8: Can multiple workflows execute simultaneously?**

A: Yes, N8N supports executing multiple workflows simultaneously. Each workflow executes independently without affecting each other.

**Q9: How to copy nodes?**

A:
1. Select node
2. Press Ctrl+C (Windows) or Cmd+C (Mac) to copy
3. Press Ctrl+V (Windows) or Cmd+V (Mac) to paste
4. Or right-click node, select "Duplicate"

**Q10: How to delete nodes?**

A:
1. Select node
2. Press Delete key
3. Or right-click node, select "Delete"
4. Confirm deletion
