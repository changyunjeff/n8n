# N8N Tutorial for Dummies Syllabus

> **教程理念**：本教程采用目标导向式教学法，每个章节围绕一个核心问题展开，通过大量实践练习帮助学习者掌握N8N工作流自动化技能。

---

## 第一章：什么是N8N？我为什么需要它？

**核心问题**：作为一个完全的新手，我如何理解N8N是什么，以及它如何解决我的实际问题？

**教学目标**：
- 理解N8N的核心概念：工作流自动化、节点、连接
- 识别N8N可以解决的典型业务场景
- 对比N8N与其他自动化工具（如Zapier、Make）的差异
- 建立对N8N应用场景的初步认知

**实践任务**：
1. **场景分析练习**：列出你日常工作中重复执行的3-5个任务
2. **工具对比研究**：创建一个对比表，分析N8N、Zapier、Make的优缺点
3. **案例研究**：阅读3个N8N实际应用案例，总结其解决的问题
4. **需求评估**：完成一份"我的自动化需求清单"，标注优先级

**评估标准**：
- 能够清晰解释N8N是什么
- 能够识别至少5个适合使用N8N的场景
- 能够说明为什么选择N8N而不是其他工具

---

## 第二章：如何从零开始安装和配置N8N？

**核心问题**：作为一个技术小白，我如何在本地或云端成功安装N8N，并进行基本配置？

**教学目标**：
- 掌握N8N的多种安装方式（Docker、npm、npx、云服务）
- 理解不同安装方式的适用场景
- 完成N8N的初始配置和账户设置
- 熟悉N8N用户界面的基本布局和功能区域

**实践任务**：
1. **安装实践**（选择一种方式）：
   - 方式A：使用npx快速启动（推荐新手）
   - 方式B：使用Docker安装（推荐有Docker经验的用户）
   - 方式C：使用npm全局安装
   - 方式D：在云平台（如DigitalOcean、AWS）部署
2. **配置练习**：
   - 创建管理员账户
   - 配置环境变量
   - 设置工作流存储位置
   - 配置Webhook URL（如需要）
3. **界面探索**：
   - 绘制N8N界面功能区域地图
   - 完成"界面导航挑战"：在30秒内找到5个关键功能
4. **故障排除**：
   - 模拟3种常见安装错误，练习解决步骤
   - 创建个人"故障排除检查清单"

**评估标准**：
- 成功安装并启动N8N
- 能够独立完成基本配置
- 能够解释每个配置选项的作用
- 能够解决至少2种常见安装问题

## 第三章：如何创建我的第一个工作流？

**核心问题**：我如何从零开始创建一个简单但完整的工作流，并理解每个步骤的作用？

**教学目标**：
- 理解工作流的基本结构：触发器→处理→输出
- 掌握节点的添加、连接和配置方法
- 学会使用工作流测试和执行功能
- 理解工作流执行日志的含义

**实践任务**：
1. **Hello World工作流**：
   - 创建一个定时触发器（Schedule Trigger）
   - 添加一个HTTP Request节点，调用一个公开API（如天气API）
   - 添加一个Set节点，提取并格式化数据
   - 添加一个Email节点，发送结果（或使用Webhook作为输出）
   - 执行并验证工作流
2. **数据流追踪练习**：
   - 创建一个包含5个节点的工作流
   - 使用"执行历史"功能追踪数据在每个节点的变化
   - 绘制数据流图，标注每个节点的输入和输出
3. **错误处理实践**：
   - 故意创建一个会失败的工作流
   - 学习阅读错误信息
   - 修复错误并重新执行
4. **工作流优化**：
   - 对同一个任务创建3种不同的工作流实现方式
   - 对比执行时间和资源消耗
   - 选择最优方案并说明理由

**评估标准**：
- 能够独立创建包含至少4个节点的工作流
- 能够解释工作流中每个节点的作用
- 能够阅读并理解执行日志
- 能够识别并修复常见的工作流错误

## 第四章：如何连接和使用第三方服务？

**核心问题**：我如何将N8N与我的日常工具（如Gmail、Slack、Google Sheets）连接起来，实现真正的自动化？

- 掌握常用服务的连接配置方法
- 学会使用Credential管理功能

**实践任务**：
1. **认证配置练习**：

   - 配置Slack Bot Token
   - 配置Google Sheets API Key
   - 配置一个需要自定义认证的服务
   - 创建个人"认证配置速查表"
2. **集成实战项目1 - 邮件自动化**：
   - 创建一个工作流：当收到特定主题的邮件时，自动提取附件并保存到Google Drive
   - 添加条件判断：根据发件人不同执行不同操作
   - 添加通知：完成后发送Slack消息
3. **集成实战项目2 - 数据同步**：
   - 创建一个工作流：每天定时从API获取数据
   - 将数据写入Google Sheets
   - 如果数据变化超过阈值，发送告警邮件
4. **集成实战项目3 - 跨平台通知**：
   - 创建一个工作流：监控网站状态
   - 如果网站宕机，同时发送邮件、Slack消息和短信（使用Twilio）
5. **Credential管理**：
   - 创建、编辑、删除Credential
   - 理解Credential的作用域和安全性
   - 练习在不同工作流间共享Credential

**评估标准**：
- 能够成功配置至少5种不同服务的认证

- 能够独立解决认证相关的常见问题

## 第五章：如何处理和转换数据？

**教学目标**：

- 学会使用N8N表达式语言
- 掌握数据聚合、过滤、排序等高级操作

**实践任务**：
1. **数据提取练习**：
   - 从一个复杂的JSON响应中提取嵌套数据
   - 使用表达式提取数组中的特定元素
   - 创建数据提取模板，可重复使用
2. **数据转换实战**：
   - 创建一个工作流：从API获取原始数据
   - 使用Set节点重命名字段
   - 使用Code节点进行数据计算和格式化
   - 使用Function节点实现复杂的数据转换逻辑
3. **数据合并项目**：
   - 从3个不同的API获取数据
   - 使用Merge节点合并数据
   - 处理数据冲突和缺失字段
   - 输出统一的JSON格式
4. **数据过滤和条件处理**：
   - 创建一个工作流：处理100条数据记录
   - 使用IF节点根据条件分流数据
   - 对满足条件的数据执行操作A，不满足的执行操作B
   - 统计每种情况的数量
5. **表达式语言深度练习**：
   - 完成20个表达式练习题（日期格式化、字符串操作、数学计算等）
   - 创建个人"表达式速查手册"
   - 实现一个复杂的表达式：组合多个数据源，进行条件判断和计算
6. **数组和批量处理**：
   - 理解Split In Batches节点的作用
   - 创建一个工作流：处理大量数据，分批处理避免超时
   - 实现错误重试机制

**评估标准**：
- 能够使用表达式提取和操作JSON数据
- 能够独立编写至少10种常用的数据转换表达式
- 能够创建包含数据合并、过滤、转换的复杂工作流
- 能够解释数据在节点间的流转过程

## 第六章：如何实现条件逻辑和循环控制？

**核心问题**：当我的工作流需要根据不同的条件执行不同的操作，或者需要重复处理多个项目时，我如何实现这些逻辑？

**教学目标**：
- 理解条件节点的使用场景
- 掌握IF节点、Switch节点的区别和用法
- 学会使用Loop节点处理数组数据
- 理解工作流的执行流程控制

**实践任务**：
1. **条件判断基础**：
   - 创建一个工作流：根据输入数据的不同值执行3条不同的路径
   - 使用IF节点实现简单条件
   - 使用Switch节点实现多分支条件
   - 对比两种方式的适用场景
2. **嵌套条件练习**：
   - 创建一个复杂的条件判断工作流
   - 实现3层嵌套的条件逻辑
   - 绘制条件决策树
   - 测试所有可能的条件组合
3. **循环处理实战**：
   - 创建一个工作流：处理一个包含100个项目的数组
   - 使用Loop Over Items节点遍历每个项目
   - 对每个项目执行相同的操作（如发送邮件）
   - 添加进度跟踪和错误处理
4. **条件+循环组合项目**：
   - 创建一个工作流：从API获取订单列表
   - 遍历每个订单
   - 根据订单状态（待处理、已发货、已完成）执行不同操作
   - 统计每种状态的订单数量
5. **错误处理和重试逻辑**：
   - 创建一个可能失败的工作流
   - 实现错误捕获和重试机制
   - 使用Continue On Fail选项
   - 创建错误通知机制
6. **工作流暂停和恢复**：
   - 创建一个需要人工审批的工作流
   - 使用Wait节点暂停执行
   - 实现审批通过/拒绝的分支逻辑

**评估标准**：
- 能够创建包含至少3层条件判断的工作流
- 能够使用循环节点处理数组数据
- 能够实现错误处理和重试逻辑
- 能够绘制并解释复杂工作流的执行流程图

## 第十二章：如何构建企业级智能合同处理自动化系统？

**核心问题**：面对企业合同处理中"附件提取→文本解析→合规校验→风险标注→报告生成→审批流转"的复杂多步骤流程，我如何运用N8N结合LangGraph、Langdock、Microsoft CoPilot等先进技术，构建一个具备可视化编排、工具联动、动态调整与高容错能力的智能自动化系统？

**教学目标**：
- 理解企业级自动化系统的架构设计原则
- 掌握N8N与LangGraph、Langdock、Microsoft CoPilot的集成方法
- 学会设计多工具协作的复杂工作流
- 掌握动态流程控制和上下文管理技术
- 理解高容错异常处理机制的设计与实现
- 学会构建智能交互和可观测性系统

**实践任务**：

1. **系统架构设计与技术选型**：
   - 分析合同处理流程的业务需求和技术挑战
   - 设计基于N8N + LangGraph + Langdock + CoPilot的系统架构
   - 绘制系统组件图和数据流图
   - 创建技术栈对比表，说明各技术的职责和优势
   - 设计工具接口规范和数据传递格式

2. **流程可视化定义（基于N8N）**：
   - 在N8N中创建合同处理主工作流，包含以下核心步骤：
     - 附件抓取节点（从指定文件夹提取合同文件，支持docx、pdf、图片格式）
     - 合同文本解析节点（根据文件格式动态选择解析工具）
     - 合规法规检索节点（基于解析后的合同条款检索相关法规）
     - 风险评估节点（分析条款与法规的冲突点，标注风险等级）
     - 报告生成节点（整合解析结果、合规结论、风险点，生成审批报告）
     - 审批流转节点（根据风险等级路由到不同审批流程）
   - 配置步骤依赖关系（如"合规检索"依赖"文本解析"的输出）
   - 实现串行和并行执行模式（如"解析合同文本"同时"提取合同关键信息"）

3. **工具注册与灵活接入（基于Langdock）**：
   - 通过Langdock注册至少4类核心工具，实现标准接口（run(input, context)）：
     - 文档解析工具（支持docx/pdf文本提取）
     - OCR解析工具（支持图片类合同识别）
     - 法规检索工具（接入Mock法规数据库）
     - 风险评估工具（分析合同文本与法规条文的冲突）
   - 实现企业信息查询API（Mock）
   - 测试工具的标准接口调用和数据返回格式
   - 创建工具注册和配置文档

4. **流程动态控制与上下文管理（基于LangGraph）**：
   - 使用LangGraph定义流程节点与状态流转
   - 实现条件分支逻辑：
     - 格式分支：根据合同文件格式（文本/图片），动态选择"文档解析工具"或"OCR工具"
     - 风险分支：高风险进入"法务人工审批"节点；中/低风险进入"自动电子审批"节点
     - 检索分支：法规检索结果不足时自动补充关键词重试，重试2次仍失败则触发CoPilot提示
   - 实现上下文管理：保存全流程中间结果，支持跨步骤数据传递
   - 实现上下文过期清理机制（流程结束后24小时自动删除）

5. **自定义合规规则实现**：
   - 实现3类合规规则检查：
     - 违约金比例检查（超过合同总金额30%为违规）
     - 诉讼时效检查（约定时效<3年或<1年为违规）
     - 争议解决方式冲突检查（同时约定仲裁和诉讼为违规）
   - 实现合规判断逻辑：无冲突或冲突有合法例外说明为合规，否则为违规
   - 实现违规处理：标注高风险，引用具体法规条文，给出修改建议，跳转至法务人工修改流程

6. **高容错异常处理机制**：
   - 实现步骤失败自动重试：配置重试次数（默认2次）和间隔（默认30秒）
   - 实现备用工具切换：主工具失败时自动切换至备用工具
   - 实现异常告警与人工介入：关键步骤连续失败后，通过CoPilot向管理员推送告警
   - 创建错误处理流程图和故障排除手册

7. **智能交互与可观测性（基于Microsoft CoPilot）**：
   - 实现CoPilot自然语言交互：
     - 支持通过自然语言触发任务（如"处理文件夹中2025年12月的销售合同"）
     - 支持查询流程状态（如"当前合同审批到哪一步了"）
     - 支持获取异常处理建议（如"解析失败该怎么办"）
   - 实现执行日志记录：展示每个步骤的输入、输出、耗时、工具调用详情、错误信息
   - 实现流程可视化：生成任务流程图（含执行路径、分支走向、节点状态），支持导出为图片

8. **端到端测试与优化**：
   - 准备测试数据集：包含不同格式的合同文件（docx、pdf、图片）
   - 测试完整流程：从附件抓取到审批流转的全流程测试
   - 测试异常场景：工具失败、网络超时、数据格式错误等
   - 性能优化：识别瓶颈节点，优化执行时间
   - 创建测试报告和性能分析文档

9. **系统部署与文档编写**：
   - 配置生产环境：环境变量、数据库、反向代理、SSL证书
   - 实现监控和告警机制
   - 编写系统架构文档
   - 编写用户操作手册
   - 编写API接口文档
   - 创建部署和维护指南

**评估标准**：
- 能够设计并实现包含至少6个核心步骤的复杂工作流
- 能够成功集成N8N、LangGraph、Langdock、Microsoft CoPilot四种技术
- 能够实现至少3种条件分支逻辑和动态流程控制
- 能够实现完整的容错机制，包括重试、备用工具切换、异常告警
- 能够实现自定义合规规则检查，准确识别违规条款
- 能够通过CoPilot实现自然语言交互和流程可视化
- 能够编写完整的技术文档和用户手册
- 系统能够稳定运行，处理至少10个不同类型的合同文件
